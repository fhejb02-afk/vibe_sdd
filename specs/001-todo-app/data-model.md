# 데이터 모델: 오늘의 할 일 웹 앱

## 엔티티: TodoItem

| 필드 | 타입 | 설명 | 제약 조건 |
|---|---|---|---|
| id | integer | 할 일 식별자 | 양의 정수, 고유값 |
| title | string | 할 일 제목 | 공백 제외 1자 이상 |
| completed | boolean | 완료 여부 | 기본값 false |
| created_at | datetime | 생성 시각 | 서버/클라이언트 기준 자동 생성 |

## 상태 전환

- 생성: 새 항목은 미완료 상태로 생성된다.
- 완료 토글: completed 값이 true/false로 바뀐다.
- 삭제: 항목이 목록에서 제거된다.

## 검증 규칙

- 제목은 비어 있거나 공백만으로 구성될 수 없다.
- 완료 상태는 boolean 값으로 관리한다.
- 삭제 요청 시 존재하지 않는 ID는 404로 처리한다.
